# `@FieldSpec`

A **default field configuration** applied to every output class generated by the spec class it is placed on. Use this when the same configuration (exclusion, nullability, transformer, or forwarded annotations) should apply uniformly across all outputs.

For output-kind-specific params (rename, validators) or when you only want to target a subset of outputs, use [`@FieldOverride`](FieldOverride.md) instead.

> **Target**: `CLASS` · **Retention**: `SOURCE` · **Repeatable**: yes
> **Priority**: `@FieldOverride` wins over `@FieldSpec` for the same `property` + suffix combination.

---

## Properties

| Property | Type | Default | Description |
|---|---|---|---|
| `property` | `String` | _(required)_ | Name of the domain property to configure. |
| `exclude` | `Boolean` | `false` | When `true`, omits this field from **every** generated output class. |
| `nullable` | `NullableOverride` | `UNSET` | Nullability override. `UNSET` preserves the domain type's nullability. See [`NullableOverride`](SupportingTypes.md#nullableoverride). |
| `transformer` | `KClass<out FieldTransformer<*, *>>` | `NoOpTransformer::class` | Transformer applied to this field in all outputs. |
| `annotations` | `Array<CustomAnnotation>` | `[]` | Annotations forwarded to this field in every output class. See [`@CustomAnnotation`](SupportingTypes.md#customannotation). |

---

## Example

### Exclude a sensitive field from all outputs

```kotlin
@FieldSpec(property = "passwordHash", exclude = true)
object UserSpec
```

### Force a field nullable in every output

```kotlin
@FieldSpec(property = "deletedAt", nullable = NullableOverride.YES)
object UserSpec
```

### Apply a transformer globally

```kotlin
@FieldSpec(
    property = "name",
    transformer = UpperCaseTransformer::class
)
object UserSpec
```

### Forward an annotation to every output's field

```kotlin
@FieldSpec(
    property = "id",
    annotations = [CustomAnnotation(
        annotation = com.fasterxml.jackson.annotation.JsonIgnore::class
    )]
)
object UserSpec
```

### Multiple defaults on the same spec

```kotlin
@FieldSpec(property = "passwordHash", exclude = true)
@FieldSpec(property = "createdAt",    nullable = NullableOverride.YES)
@FieldSpec(property = "name",         transformer = UpperCaseTransformer::class)
object UserSpec
```

---

## Precedence

When both `@FieldSpec` and `@FieldOverride` configure the same property:

```
@FieldOverride(for_ = ["Entity"], property = "name", rename = "userName")     // rename only for Entity
@FieldSpec(property = "name", transformer = UpperCaseTransformer::class)      // applies across outputs
```

`@FieldOverride` values win for every parameter where the `@FieldOverride` provides a non-default value.
