# `@ClassField`

A **shared field override** applied to every output class generated by the spec class it is placed on. Use this when the same configuration (exclusion, nullability, transformer, or forwarded annotations) should apply uniformly across all outputs.

For output-kind-specific params (column name, field rename, validation rules) or when you only want to target a subset of outputs, use [`@FieldSpec`](FieldSpec.md) instead.

> **Target**: `CLASS` · **Retention**: `SOURCE` · **Repeatable**: yes  
> **Priority**: `@FieldSpec` wins over `@ClassField` for the same `property` + suffix combination.

---

## Properties

| Property | Type | Default | Description |
|---|---|---|---|
| `property` | `String` | _(required)_ | Name of the domain property to configure. |
| `exclude` | `Boolean` | `false` | When `true`, omits this field from **every** generated output class. |
| `nullable` | `NullableOverride` | `UNSET` | Nullability override. `UNSET` preserves the domain type's nullability. See [`NullableOverride`](SupportingTypes.md#nullableoverride). |
| `transformer` | `KClass<out FieldTransformer<*, *>>` | `NoOpTransformer::class` | Transformer applied to this field in all outputs. |
| `transformerRef` | `String` | `""` | Named transformer from a [`@TransformerRegistry`](TransformerRegistry.md). Wins over `transformer` when both are set. |
| `annotations` | `Array<CustomAnnotation>` | `[]` | Annotations forwarded to this field in every output class. See [`@CustomAnnotation`](RuleExpression.md#customannotation). |

---

## Example

### Exclude a sensitive field from all outputs

```kotlin
@ClassField(property = "passwordHash", exclude = true)
object UserSpec
```

### Force a field nullable in every output

```kotlin
@ClassField(property = "deletedAt", nullable = NullableOverride.YES)
object UserSpec
```

### Apply a transformer globally

```kotlin
@ClassField(
    property = "name",
    transformerRef = "upperCase"   // registered in @TransformerRegistry
)
object UserSpec
```

### Forward an annotation to every output's field

```kotlin
@ClassField(
    property = "id",
    annotations = [CustomAnnotation(
        annotation = com.fasterxml.jackson.annotation.JsonIgnore::class
    )]
)
object UserSpec
```

### Multiple overrides on the same spec

```kotlin
@ClassField(property = "passwordHash", exclude = true)
@ClassField(property = "createdAt",    nullable = NullableOverride.YES)
@ClassField(property = "name",         transformerRef = "upperCase")
object UserSpec
```

---

## Precedence

When both `@ClassField` and `@FieldSpec` configure the same property:

```
@FieldSpec(for_ = ["Entity"], property = "name", column = "user_name")  // also sets transformer here
@ClassField(property = "name", transformerRef = "upperCase")             // overridden for Entity suffix

// Result for Entity: column = "user_name", transformer from @FieldSpec (if set), else ClassField's
```

`@FieldSpec` values win for every parameter where the `@FieldSpec` provides a non-default value.
